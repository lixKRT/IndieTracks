## 二、数据源与 XPath / API 映射

### 2.1 首页 API — 专辑列表和社团基础信息

```
GET https://www.dizzylab.net/apis/getdiscs/?l={start}&r={end}&sort=ad&type=album
```

| JSON 字段 | 数据库字段 | 说明 |
|:---|:---|:---|
| `id` | `albums.dizzylab_id` | 如 `"SW20"` |
| `title` | `albums.title` | `"冬日咏叙调"` |
| `price` | `albums.price` | `40.0` |
| `cover` | `albums.cover_url` | CDN 完整 URL |
| `label` | `circles.name` | `"Static World"` |
| `labelid` | `circles.dizzylab_labelid` | `30` |
| `labelcover` | `circles.logo_url` | |
| `tags[]` | `tags.name`（去除前导 `#`） | `["#纯音乐","Static World","#东方project"]` |

翻页：`l` 起始位置，`r` 结束位置。`l=0&r=24` → 第 1 批，`l=24&r=48` → 第 2 批。

### 2.2 专辑详情页 HTML — `/d/{slug}/`

| 数据库字段 | XPath / 取值规则 | 已验证样例 |
|:---|:---|:---|
| `albums.title` | `//title/text()` 去掉末尾 `" - dizzylab"` | `"冬日咏叙调"` |
| `albums.info_title` | 所有 `<p>` 段落文本，`\n` 拼接 | 专辑描述长文 |
| `albums.info_content` | 所有 `<h3 class='text-left'>` 文本，`\n` 拼接 | "Track List" 等区段标题 |
| `albums.publish_date` | 匹配文本 `"发布于(\d{4})年(\d{1,2})月(\d{1,2})日"` | `"2026-05-01"` |
| `albums.cover_url` | `//img[@id='imgsrc0']/@data-src`（懒加载） | CDN URL |

#### 2.2.1 曲目列表（播放控件）

**容器**：`//ul[contains(@class,'playlist--list')]/li`

每个 `<li>` 结构：
```html
<li data-id="0" data-audio="https://streaming.dizzylab.net/.../preview/1.mp3">
  <span class="t-title">1. 冬临之时 - Static World (02:48)</span>
</li>
```

| 数据库字段 | XPath | 解析规则 |
|:---|:---|:---|
| `sort_order` | `li/@data-id` | `data-id + 1`（0-based → 1-based） |
| `file_name` | `li//span[@class='t-title']/text()` | 去掉序号前缀，正则提取曲名部分 |
| `track_length` | 同上 | 正则提取 `(MM:SS)` |
| `object_key` | `li/@data-audio` | **暂不下载**，仅采集 URL 备用 |

解析示例：`"1. 冬临之时 - Static World (02:48)"`
→ `file_name = "冬临之时 - Static World"`，`track_length = "02:48"`

#### 2.2.2 标签

**XPath**：`//h4[@class='text-left']/a/text()`

原始文本：`"##纯音乐"` `"#Static World"` `"#治愈系"` `"##东方project"`
处理：`strip("#")` → `"纯音乐"` `"Static World"` `"治愈系"` `"东方project"`

#### 2.2.3 支持的人（已购买）— AJAX API

```
GET https://www.dizzylab.net/albums/getbuyers/?discid={slug}&l={start}&r={end}
```

| JSON 字段 | 数据库 | 说明 |
|:---|:---|:---|
| `names[]` | `users.username` | |
| `user_url[]` | `users.dizzylab_user_id` | 从 `/albums/u/90162` 提取数字 |
| `avatar_url[]` | `users.avatar_url` | 去外层引号 |

#### 2.2.4 评论 — AJAX API

```
GET https://www.dizzylab.net/albums/getdisccomment/?discid={slug}&l={start}&r={end}
```

| JSON 字段 | 数据库 | 说明 |
|:---|:---|:---|
| `disc_comment[]` | `comments.content` | |
| `user_names[]` | 关联 `users` | |
| `user_url[]` | `users.dizzylab_user_id` | |
| `avatar_url[]` | `users.avatar_url` | |

### 2.3 社团详情页 HTML — `/l/{name}/`

| 数据库字段 | XPath | 说明 |
|:---|:---|:---|
| `circles.name` | 页面标题 | |
| `circles.description` | 社团介绍长文区域 | |
| `circles.logo_url` | `//img`（社团页头像） | |

#### 2.3.1 社团成员

**XPath**：`//p[text()='成员']/following-sibling::div//a[contains(@href, '/u/')]`

```html
<a href='/u/4725' title='...<br>Static World'>
  <img src="..." />
</a>
```

- `href='/u/4725'` → `dizzylab_user_id = 4725`
- `title` 属性中 `<br>` 后的文本 → `username`

### 2.4 Setup 页 — 用户角色

```
GET https://www.dizzylab.net/setup
```

**STAFF 用户**：
- 标题：`//h2[contains(., 'STAFF')]`
- 用户链接：`//h2[contains(., 'STAFF')]/following-sibling::a[contains(@href, '/u/')]`
- `href='/u/1'` → `user_id = 1`，`user_role = 'staff'`

**PRO 用户**：
- 标题：`//h2[contains(., 'PRO')]`
- 同上逻辑，`user_role = 'pro'`

### 2.5 用户维页面（无需登录）

| 页面 | URL | 对应表 | 说明 |
|:---|:---|:---|:---|
| 已购专辑 | `/u/{id}/music/` | `owned_albums` | 可通过 getbuyers API 替代 |
| 收藏 | `/u/{id}/likes/` | `favorites` | 用户喜欢的专辑列表 |
| 关注 | `/u/{id}/following/` | `circle_follows` | 用户关注的社团列表 |

## 三、延迟策略配置

### 3.1 配置文件 `crawler/config/delay.json`

```json
{
  "active": "default",
  "strategies": {
    "default": {
      "download_delay": 3,
      "between_albums_min": 60,
      "between_albums_random": 60,
      "concurrent_requests_per_domain": 1
    },
    "gentle": {
      "download_delay": 5,
      "between_albums_min": 120,
      "between_albums_random": 120,
      "concurrent_requests_per_domain": 1
    },
    "test": {
      "download_delay": 1,
      "between_albums_min": 10,
      "between_albums_random": 10,
      "concurrent_requests_per_domain": 1
    }
  }
}
```

**字段说明：**

| 字段 | 含义 | Scrapy 对应 |
|:---|:---|:---|
| `download_delay` | 同一域名连续请求的最小间隔（秒） | `DOWNLOAD_DELAY` |
| `between_albums_min` | 每张专辑全部子请求完成后，等待的最小秒数 | Spider 层 `time.sleep()` |
| `between_albums_random` | 额外叠加随机 0~N 秒 | `random.randint(0, N)` |
| `concurrent_requests_per_domain` | 同域名最大并发请求数 | `CONCURRENT_REQUESTS_PER_DOMAIN` |

### 3.2 数据库配置 `crawler/config/database.json`

```json
{
  "host": "localhost",
  "port": 5432,
  "database": "IndieTracks",
  "user": "",
  "password": ""
}
```

> **注意**：`user` 和 `password` 自行填写，不要提交到 Git。

## 五、`album_test.py` 爬虫流程

### 5.1 入口

```python
start_urls = ["https://www.dizzylab.net/apis/getdiscs/?l=0&r=10&sort=ad&type=album"]
```

取 10 条 disc，逐张处理。

### 5.2 单张专辑处理链（串行）

```
parse_disc_list(response)
  → 遍历 10 条 disc
  → 对每条 disc:
      1. GET /d/{slug}/  → parse_album_detail
         │
         ├─ 解析 HTML → yield AlbumItem, WorkFileItem[], TagItem[], AlbumTagItem[]
         ├─ 同时 yield Request:
         │   ┌ GET /albums/getbuyers/?discid={slug}&l=0&r=60    → parse_buyers
         │   ├ GET /albums/getdisccomment/?discid={slug}&l=0&r=10 → parse_comments
         │   └ GET /l/{label_name}/                               → parse_circle_detail
         │        └─ 解析 HTML → yield CircleItem, UserCircleItem[]
         │
      2. 等待专辑间延迟: sleep(60 + random(0, 60))
      3. 处理下一张
```

### 5.3 Pipeline 写入顺序

| 步骤 | 表 | 去重方式 |
|:---|:---|:---|
| 1 | `tags` | `ON CONFLICT (name) DO NOTHING`，返回已有 `tag_id` |
| 2 | `circles` | `ON CONFLICT (dizzylab_labelid) DO NOTHING` |
| 3 | `users` | `ON CONFLICT (dizzylab_user_id) DO NOTHING` |
| 4 | `albums` | `ON CONFLICT (dizzylab_id) DO NOTHING`，RETURNING `album_id` |
| 5 | `album_tags` | `ON CONFLICT (album_id, tag_id) DO NOTHING` |
| 6 | `album_circles` | `ON CONFLICT (album_id, circle_id) DO NOTHING` |
| 7 | `work_files` | `DELETE + INSERT`（按 album_id 清旧数据再插） |
| 8 | `comments` | 按 `(album_id, user_id, created_at)` 去重 |
| 9 | `owned_albums` | `ON CONFLICT (user_id, album_id) DO NOTHING` |
| 10 | `user_circles` | `ON CONFLICT (user_id, circle_id) DO NOTHING` |

### 5.4 XPath 解析速查表（爬虫内用）

| 目标 | 表达式 |
|:---|:---|
| 曲目 `<li>` 列表 | `//ul[contains(@class,'playlist--list')]/li` |
| 曲目 data-id | `./@data-id` |
| 曲目 data-audio | `./@data-audio` |
| 曲目标题 span | `.//span[@class='t-title']/text()` |
| 封面 data-src | `//img[@id='imgsrc0']/@data-src` |
| 标签 `<a>` 列表 | `//h4[@class='text-left']/a/text()` |
| 发布日期 | 正则 `发布于(\d{4})年(\d{1,2})月(\d{1,2})日` |
| 社团成员 | `//p[text()='成员']/following-sibling::div//a[contains(@href, '/u/')]` |
| 成员 user_id | `./@href` → 正则 `/u/(\d+)` |

## 六、爬虫执行命令

**首次运行前：**
```bash
# 1. 编辑 crawler/config/database.json，填入 PostgreSQL 用户名和密码
# 2. 创建数据库（如果未创建）
cd crawler
env\Scripts\activate
python -c "import psycopg2; c=psycopg2.connect(host='localhost',port=5432,user='postgres',password='你的密码',database='postgres'); c.autocommit=True; c.cursor().execute('CREATE DATABASE indietracks'); print('OK')"
# 3. 建表
python setup_db.py
```

**爬虫运行：**
```bash
cd crawler
env\Scripts\activate

# 测试爬虫（10 张专辑，约 1.5-2 小时）
scrapy crawl album_test

# 增量爬虫（后续）
scrapy crawl album_incremental
```

**重置数据（清空重来）：**
```bash
psql -U postgres -d indietracks -f database/init.sql
```
