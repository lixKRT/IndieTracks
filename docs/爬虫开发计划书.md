## 二、数据源与 XPath / API 映射

### 2.1 首页 API — 专辑列表和社团基础信息

```
GET https://www.dizzylab.net/apis/getdiscs/?l={start}&r={end}&sort=ad&type=album
```

| JSON 字段 | 数据库字段 | 说明 |
|:---|:---|:---|
| `id` | `albums.dizzylab_id` | 如 `"SW20"` |
| `title` | `albums.title` | `"冬日咏叙调"` |
| `price` | `albums.price` | `40.0` |
| `cover` | `albums.cover_url` | CDN 完整 URL |
| `label` | `circles.name` | `"Static World"` |
| `labelid` | `circles.dizzylab_labelid` | `30` |
| `labelcover` | `circles.logo_url` | |
| `tags[]` | `tags.name`（去除前导 `#`） | `["#纯音乐","Static World","#东方project"]` |

翻页：`l` 起始位置，`r` 结束位置。`l=0&r=24` → 第 1 批，`l=24&r=48` → 第 2 批。

### 2.2 专辑详情页 HTML — `/d/{slug}/`

| 数据库字段 | XPath / 取值规则 | 已验证样例 |
|:---|:---|:---|
| `albums.title` | `//title/text()` 去掉末尾 `" - dizzylab"` | `"冬日咏叙调"` |
| `albums.info_title` | 所有 `<p>` 段落文本，`\n` 拼接 | 专辑描述长文 |
| `albums.info_content` | 所有 `<h3 class='text-left'>` 文本，`\n` 拼接 | "Track List" 等区段标题 |
| `albums.publish_date` | 匹配文本 `"发布于(\d{4})年(\d{1,2})月(\d{1,2})日"` | `"2026-05-01"` |
| `albums.cover_url` | `//img[@id='imgsrc0']/@data-src`（懒加载） | CDN URL |

#### 2.2.1 曲目列表（播放控件）

**容器**：`//ul[contains(@class,'playlist--list')]/li`

每个 `<li>` 结构：
```html
<li data-id="0" data-audio="https://streaming.dizzylab.net/.../preview/1.mp3">
  <span class="t-title">1. 冬临之时 - Static World (02:48)</span>
</li>
```

| 数据库字段 | XPath | 解析规则 |
|:---|:---|:---|
| `sort_order` | `li/@data-id` | `data-id + 1`（0-based → 1-based） |
| `file_name` | `li//span[@class='t-title']/text()` | 去掉序号前缀，正则提取曲名部分 |
| `track_length` | 同上 | 正则提取 `(MM:SS)` |
| `object_key` | `li/@data-audio` → 下载 → 上传 MinIO | `audio/preview/{slug}/{sort_order:03d}.mp3` |

解析示例：`"1. 冬临之时 - Static World (02:48)"`
→ `file_name = "冬临之时 - Static World"`，`track_length = "02:48"`

#### 2.2.2 标签

**XPath**：`//h4[@class='text-left']/a/text()`

原始文本：`"##纯音乐"` `"#Static World"` `"#治愈系"` `"##东方project"`
处理：`strip("#")` → `"纯音乐"` `"Static World"` `"治愈系"` `"东方project"`

#### 2.2.3 支持的人（已购买）— AJAX API

```
GET https://www.dizzylab.net/albums/getbuyers/?discid={slug}&l={start}&r={end}
```

| JSON 字段 | 数据库 | 说明 |
|:---|:---|:---|
| `names[]` | `users.username` | |
| `user_url[]` | `users.dizzylab_user_id` | 从 `/albums/u/90162` 提取数字 |
| `avatar_url[]` | `users.avatar_url` | 去外层引号 |

#### 2.2.4 评论 — AJAX API

```
GET https://www.dizzylab.net/albums/getdisccomment/?discid={slug}&l={start}&r={end}
```

| JSON 字段 | 数据库 | 说明 |
|:---|:---|:---|
| `disc_comment[]` | `comments.content` | |
| `user_names[]` | 关联 `users` | |
| `user_url[]` | `users.dizzylab_user_id` | |

#### 2.2.5 喜欢数 — AJAX API
`GET /albums/getthisdisclikes/?discid={slug}` → `{"likes": N}`。仅计数。

#### 2.2.6 长评 REPO — AJAX API

### 2.3 社团列表 — AJAX API

```
GET https://www.dizzylab.net/apis/getlabels/?l={start}&r={end}&sort=ad
```

| JSON 字段 | 数据库字段 | 说明 |
|:---|:---|:---|

| `title` | `circles.name` | |
| `labelcover` | `circles.logo_url` | Logo CDN URL |
| `description` | `circles.description` | 社团简介 |
| `discs[]` | — | 社团发布的专辑 slug 列表 |

### 2.4 社团详情页 HTML — `/l/{name}/`

| 数据库字段 | XPath | 说明 |
|:---|:---|:---|
| `circles.name` | 页面标题 | |
| `circles.description` | 社团介绍长文区域 | |
| `circles.logo_url` | `//img`（社团页头像） | |

#### 2.4.1 社团成员

**XPath**：`//p[text()='成员']/following-sibling::div//a[contains(@href, '/u/')]`

```html
<a href='/u/4725' title='...<br>Static World'>
  <img src="..." />
</a>
```

- `href='/u/4725'` → `dizzylab_user_id = 4725`
- `title` 属性中 `<br>` 后的文本 → `username`

### 2.5 Setup 页 — 用户角色

```
GET https://www.dizzylab.net/setup
```

**STAFF 用户**：
- 标题：`//h2[contains(., 'STAFF')]`
- 用户链接：`//h2[contains(., 'STAFF')]/following-sibling::a[contains(@href, '/u/')]`
- `href='/u/1'` → `user_id = 1`，`user_role = 'staff'`

**PRO 用户**：
- 标题：`//h2[contains(., 'PRO')]`
- 同上逻辑，`user_role = 'pro'`

### 2.6 用户维页面（无需登录）

**数据源：** 三个 HTML 页面，翻页参数不统一：

| 页面 | URL 格式 | 分页参数 | 对应表 | 容器 id |
|:---|:---|:---|:---|:---|
| 已购专辑 | `/u/{id}/music/` | `?page=N` | `owned_albums` | `id="discs"` |
| 收藏 | `/u/{id}/likes/` | `?dp=N` | `favorites` | `id="album"` |
| 关注社团 | `/u/{id}/following/` | `?dp=N` | `circle_follows` | `id="album"` |

**已购 / 收藏页面解析：**
- 专辑卡片：`//div[@id='discs' or @id='album']//a[contains(@href, '/d/')]`
- `href="/d/{slug}"` → `dizzylab_id`
- 标题：`ancestor::div[contains(@class,'card')]//p[contains(@class,'text-truncate')]/text()`
- 分页：`//ul[contains(@class,'pagination')]//a[contains(@href,'page=') or contains(@href,'dp=')]/@href`

**关注页面解析：**
- 社团卡片：`//div[contains(@class,'tab-pane')]//a[contains(@href, '/l/')]`
- `href="/l/{name}"` → URL 解码得社团名 → DB 查询 `circle_id`
- 页面仅含社团关注，无用户关注（dizzylab 不支持用户间关注）

## 三、配置文件体系

### 3.1 延迟策略 `crawler/config/delay.json`

```json
{
  "active": "default",
  "strategies": {
    "default": {
      "download_delay": 3,
      "between_albums_min": 60,
      "between_albums_random": 60,
      "between_tracks_min": 2,
      "between_tracks_random_max": 5,
      "concurrent_requests_per_domain": 1
    },
    "gentle": {
      "download_delay": 5,
      "between_albums_min": 120,
      "between_albums_random": 120,
      "between_tracks_min": 3,
      "between_tracks_random_max": 10,
      "concurrent_requests_per_domain": 1
    },
    "test": {
      "download_delay": 1,
      "between_albums_min": 10,
      "between_albums_random": 10,
      "between_tracks_min": 1,
      "between_tracks_random_max": 3,
      "concurrent_requests_per_domain": 1
    }
  }
}
```

| 字段 | 含义 | 生效位置 |
|:---|:---|:---|
| `download_delay` | 同域名请求最小间隔（秒） | `settings.py` → `DOWNLOAD_DELAY` |
| `between_albums_min` | 专辑全部子请求完成后等待的最小秒数 | 专辑类爬虫（`parse_disc_list` 循环内） |
| `between_albums_random` | 额外叠加 0~N 秒随机 | 同上 |
| `between_tracks_min` | 曲目间下载固定延迟（秒） | `parse_album_detail` 曲目循环内 |
| `between_tracks_random_max` | 曲目间额外叠加 0~N 秒随机 | 同上 |
| `concurrent_requests_per_domain` | 同域名最大并发 | `settings.py` → `CONCURRENT_REQUESTS_PER_DOMAIN` |

**所有爬虫必须遵守此配置**：专辑类爬虫使用 `download_delay` + `between_albums_sleep` + `between_tracks`，非专辑类爬虫仅使用 `download_delay`。

### 3.2 数据库配置 `crawler/config/database.json`

```json
{
  "host": "localhost",
  "port": 5432,
  "database": "indietracks",
  "user": "postgres",
  "password": "请自行填写"
}
```

> **user 和 password 自行填写，不要提交到 Git。**

### 3.3 爬虫行为控制 `crawler/config/spider.json`

```json
{
  "mode": "full",
  "max_albums": 50,
  "max_users": 20
}
```

| 字段 | 值 | 含义 |
|:---|:---|:---|
| `mode` | `"full"` | 全量模式：已存在的专辑/社团/用户数据也会重新爬取，作为数据刷新 |
| `mode` | `"incremental"` | 增量模式：跳过数据库中已有的记录，只入库新数据 |
| `max_albums` | 正整数（如 `50`） | 专辑类爬虫最多处理多少张专辑（达到后停止） |
| `max_albums` | `0` | 不限额，爬完所有可访问数据 |
| `max_users` | 正整数（如 `20`） | `user_pages` 最多处理多少个用户 |
| `max_users` | `0` | 不限额，遍历所有用户 |

**适用范围：** `album_bulk`、`circle_members`、`user_roles`、`user_pages` 遵守此配置。
**例外：** `album_incremental` 不遵守此配置（永远增量、永远不限额）。
**例外：** `album_test` 不遵守此配置（调试用，仅爬 10 张）。

`max_albums` / `max_users` 计数器每次启动从 0 开始累计。默认值 50 / 20 用于首次全量铺底。后续日常同步可切 `"incremental"` 并调整限额。

### 3.4 MinIO 连接配置 `crawler/config/minio.json`

```json
{
  "endpoint": "http://localhost:9000",
  "access_key": "",
  "secret_key": "",
  "bucket": "indietracks",
  "prefixes": {
    "audio_preview": "audio/preview/",
    "audio_full": "audio/full/",
    "covers": "covers/",
    "logos": "logos/",
    "avatars": "avatars/"
  }
}
```

文件分类策略：单个 bucket `indietracks` 内 5 个前缀。需要新增类型时加前缀即可，不新增 bucket。`access_key` 和 `secret_key` 由 `setup-minio.bat` 自动回填。

## 五、爬虫设计文档

### 5.1 `album_test.py` — 测试爬虫

**用途：** 开发调试，验证数据结构正确性，不用于正式数据采集。

- 不受 `spider.json` 控制，固定爬 API 返回的前 10 张专辑。
- 子请求链路：album detail → buyers → comments → circle_detail。
- 遵守 `delay.json`（当前 active: "test"）。

### 5.2 `album_bulk.py` — 批量爬虫

**用途：** 首次数据铺底 + 后续全量刷新。最常用的爬虫。

**控制逻辑（遵守 `spider.json`）：**

| mode | dizzylab_id 已存在 | max_albums 计数 |
|:---|:---|:---|
| `"full"` | 正常重爬（生成全量子请求） | 计入，累计到 `max_albums` 停止 |
| `"incremental"` | 跳过该专辑，不生成子请求 | 不计入，仅新专辑计 |

**停止条件：**

- 累计处理专辑数达到 `max_albums`（`max_albums > 0`）
- API 翻页返回 0 条（数据源耗尽）

**子请求链路（同 album_test）：**
1. `parse_disc_list` → API 翻页，逐条取 disc
2. `parse_album_detail` → HTML 解析 → yield Album、Tags、Circle
3. 曲目列表遍历 → 逐条 `download_and_upload()` 下载音频并上传 MinIO → yield WorkFile（object_key=MinIO 路径）
4. 并行子请求：`parse_buyers`、`parse_comments`、`parse_circle_detail`
5. 专辑间延迟：`between_tracks` 曲目间随机延迟 → 专辑子请求完成后 `between_albums_sleep()`

**进度日志格式：** `[已抓/限额] slug - 标题 ✓ | mode | 跳过: N`

| mode | 跳过计数 |
|:---|:---|
| `"full"` | 始终为 0（全量重爬，不跳过任何专辑） |
| `"incremental"` | 累计跳过数量（已有专辑逐张 +1） |

> 例：`[12/50] SW20 - 冬日咏叙调 ✓ | mode=full | 跳过: 0`
> 例：`[12/50] SW20 - 冬日咏叙调 ✓ | mode=incremental | 跳过: 3`

### 5.3 `album_incremental.py` — 增量爬虫

**用途：** 定期调度，追平最新发布的专辑。**不受 `spider.json` 控制。**

- 从 API 第一页（`l=0`）开始，按发布时间倒序。
- 遇到第一条数据库中已存在的 `dizzylab_id` 立即停止整个爬虫。
- 不做 `between_albums_sleep`（通常只有 1-3 张新专辑），但遵守 `download_delay` 和 `between_tracks`。
- 子请求链路同 album_bulk（含音频下载+MinIO 上传）。

### 5.4 `circle_members.py` — 社团成员爬虫

**用途：** 为 `circles` 表中的社团补充成员列表。

**控制逻辑（遵守 `spider.json`）：**

| mode | 行为 |
|:---|:---|
| `"full"` | 从 DB 读 circles 表全部社团，逐一爬成员页，重写 user_circles |
| `"incremental"` | 跳过已有成员的社团（社团下已有 UserCircle 记录则整社跳过） |

**流程：**
1. 从数据库 `SELECT circle_id, name FROM circles`
2. 逐社团爬 `https://www.dizzylab.net/l/{name}/`
3. 解析成员列表 → yield UserItem + UserCircleItem
4. 每社团间隔 `download_delay` 秒

### 5.5 `user_roles.py` — 用户角色爬虫

**用途：** 从 `/setup` 页获取 STAFF/PRO 用户列表，填充 `users.user_role`。

**控制逻辑（遵守 `spider.json`）：**

| mode | 行为 |
|:---|:---|
| `"full"` | 爬 /setup，所有解析的 STAFF/PRO 用户全部 UPDATE role |
| `"incremental"` | 仅 UPDATE 当前 role 为空（null/'normal'）的用户 |

**流程：**
1. 爬 `https://www.dizzylab.net/setup`
2. 解析 STAFF 区域 → 所有用户 `user_role = 'staff'`
3. 解析 PRO 区域 → 所有用户 `user_role = 'pro'`
4. 直接 DB UPDATE，不走 Pipeline

### 5.6 `user_pages.py` — 用户维页面爬虫

**用途：** 爬取用户的已购专辑、收藏、社团关注三个维页面。

**控制逻辑（遵守 `spider.json`）：**

| mode | 行为 |
|:---|:---|
| `"full"` | 忽略 `userpage_crawled_at`，所有用户重爬 |
| `"incremental"` | 跳过 `userpage_crawled_at` 距今 < 30 天的用户 |

**用户来源（双模式）：**
1. 命令行指定：`scrapy crawl user_pages -a user_ids=2,3,5`
2. DB 读取：`SELECT dizzylab_user_id, username FROM users ORDER BY dizzylab_user_id`（incremental 模式过滤 30 天内已爬用户）

**流程：**
1. 获取用户列表 → 每用户发起 3 个首页请求（music/likes/following）
2. `parse_music` → 解析专辑卡片 → DB 查 album_id → yield OwnedAlbumItem → 翻页 `?page=N`
3. `parse_likes` → 同 music 解析 → yield FavoriteItem → 翻页 `?dp=N`
4. `parse_following` → 解析社团卡片 → DB 查 circle_id → yield CircleFollowItem → 翻页 `?dp=N`
5. 爬完所有用户后批量 `UPDATE userpage_crawled_at`

**DB 辅助查询（不在 Pipeline 做）：**
- `_resolve_album_id(slug)` → `SELECT album_id FROM albums WHERE dizzylab_id = %s`（不存在则 INSERT 占位行）
- `_resolve_circle_id(name)` → `SELECT circle_id FROM circles WHERE name = %s`
- `_resolve_user_db_id(dizzylab_uid)` → `SELECT user_id FROM users WHERE dizzylab_user_id = %s`

**进度日志格式：** `[已处理] username (uid=X) | mode | 跳过: N`

## 六、Pipeline 写入策略

| 步骤 | 表 | 去重方式 |
|:---|:---|:---|
| 1 | `tags` | `ON CONFLICT (name) DO UPDATE ... RETURNING tag_id` |
| 2 | `circles` | `ON CONFLICT (dizzylab_labelid) DO UPDATE ... RETURNING circle_id` |
| 3 | `users` | `ON CONFLICT (dizzylab_user_id) DO UPDATE ... RETURNING user_id` |
| 4 | `albums` | `ON CONFLICT (dizzylab_id) DO UPDATE ... RETURNING album_id` |
| 5 | `album_tags` | `ON CONFLICT DO NOTHING` |
| 6 | `album_circles` | `ON CONFLICT DO NOTHING` |
| 7 | `work_files` | `DELETE + INSERT`（按 album_id 清旧数据再插） |
| 8 | `comments` | 直接 INSERT |
| 9 | `owned_albums` | `ON CONFLICT DO NOTHING` |
| 10 | `user_circles` | `ON CONFLICT DO NOTHING` |
| 11 | `favorites` | `ON CONFLICT DO NOTHING` |
| 12 | `circle_follows` | `ON CONFLICT DO NOTHING` |
| 13 | `user_follows` | `ON CONFLICT DO NOTHING`（预留，dizzylab 无此功能） |

> 共处理 13 种 Item（14 种定义，`UserFollowItem` 预留）。Pipeline 通过 `_dizzylab_*` 临时字段解析外键引用，在内存缓存中查找 db 主键。

## 七、MinIO 部署架构

### 7.1 文件分类

单 bucket `indietracks`，5 个前缀：

```
indietracks/               ← 一个 bucket 替代五个
├── audio/preview/         # 试听音频 (MP3) — 已集成到专辑爬虫
│   └── {album_slug}/
│       └── {sort_order:03d}.mp3
├── audio/full/            # 完整版音频 (FLAC / WAV) — 预留
├── covers/                # 专辑封面图 — 后续
├── logos/                 # 社团 logo — 后续
└── avatars/               # 用户头像 — 后续
```

### 7.2 音频下载流程

专辑爬虫（`album_bulk` / `album_incremental` / `album_test`）在 `parse_album_detail` 中：

1. 解析曲目列表 `<li data-audio="CDN URL">`
2. 调用 `utils/minio.py::download_and_upload(cdn_url, album_slug, sort_order)`
   - `requests.get()` 下载 MP3
   - `minio-py` `put_object()` 上传到 `audio/preview/{slug}/{sort_order:03d}.mp3`
   - 返回 `(object_key, file_size)` 或失败返回 `None`（失败跳过该曲目）
3. 填入 `WorkFileItem.object_key`（MinIO 路径）和 `file_size`（真实字节数）
4. 曲目间延迟：`between_tracks_min + rand(0, between_tracks_random_max)` 秒

### 7.3 一键部署脚本

`scripts/windows/setup-minio.bat` 执行以下步骤：

1. 检查 `tools/minio/minio.exe` → 不存在则自动从 MinIO 官方源下载
2. 启动 MinIO 服务（端口 9000，数据目录 `database/minio-data/`，控制台端口 9001）
3. 创建 bucket `indietracks`
4. 创建应用用户 `indietracks_app` + 读写策略
5. 将连接信息写入 `crawler/config/minio.json`
