## 三、组件分层

### 3.1 分子（molecules）
原子组合，有独立业务语义，可在不同有机体和页面中复用。

### 3.2 有机体（organisms）
完整功能区块，由分子拼装而成，通常有专属业务逻辑。

### 3.3 布局（layouts）
页面骨架。`MainLayout` 包含：Navbar（顶） + `<router-view />`（中） + PlayerBar（底） + Footer（底）。

### 3.4 页面（views）
路由入口，负责注入数据给有机体。组件结构本身是无状态的，数据从 Pinia store 或 Mock 数据获取。

## 四、路由表

| 路由 | 组件 | 说明 |
|:---|:---|:---|
| `/` | Home | 首页，专辑网格，按发布日期倒序 |
| `/album/:id` | AlbumDetail | 专辑详情 |
| `/labels` | Labels | 社团列表 |
| `/label/:id` | LabelDetail | 社团详情 |
| `/tag` | TagBrowse | 分类浏览，支持 `?tag=xxx`、`?search=xxx`、`?price=free\|paid` |

## 五、Pinia Store 设计

### 5.1 player store

唯一 store，管理播放器全局状态。

```js
// stores/player.js
{
  playlist: [             // 播放列表（支持拖拽排序）
    {
      file_id: 1,
      file_name: "Transcendence of Fluid Neon",
      duration: "4:46",
      album_title: "Touhou Colours：Aquamarine",
      artist: "Chroma_Sounds虹彩",
      preview_url: null  // Mock 阶段为 null，后端联调后为预签名 URL
    }
  ],
  current_index: 0,       // 当前播放曲目在 playlist 中的索引
  is_playing: false,
  is_expanded: false      // 底部条 / 展开面板
}
```

**行为规则：**
- 用户在任意专辑详情页点击试听按钮 → 将该专辑的所有 `preview` 曲目追加至 `playlist` 末尾，从点击的那首开始播。
- 拖拽排序 `playlist` → 即时生效。
- 刷新页面 → 从 localStorage 恢复 `playlist` 和 `current_index`。
- 不在 store 中管理认证态。Mock 阶段不模拟登录。

## 七、全局样式策略

### 7.1 文件拆分

| 文件 | 内容 |
|:---|:---|
| `tokens.css` | CSS 自定义属性：颜色、间距、字体、圆角、过渡时间 |
| `reset.css` | `*` 重置 + `body` 基础 |
| `utilities.css` | `.container`、`.loading-spinner`、modal 通用样式 |

### 7.2 tokens.css 示例

```css

  --color-border: #222;
  --color-border-hover: #333;
```

## 八、响应式设计

### 8.1 断点

| 断点 | 视口宽度 | Grid 列数 |
|:---|:---|:---|
| 手机 | < 640px | 2 列 |
| 平板 | 640px - 1024px | 3 列 |
| 桌面 | > 1024px | 4 列 |

### 8.2 专辑卡片

- 封面：`aspect-ratio: 1 / 1`，图片 `object-fit: cover`。
- 信息区：固定 `min-height`，保证网格内所有卡片高度一致。
- 标题溢出：`line-clamp: 2`，标签溢出：`line-clamp: 1`。

### 8.3 导航栏

- 手机：Logo + 汉堡菜单（搜索折叠到菜单内）。
- 桌面：Logo + 导航链接 + 搜索框（展开）。
